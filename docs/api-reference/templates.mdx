# Templates API

Retrieve approved message templates from your Meta Business account. Templates are managed in the [Meta Business Manager](https://business.facebook.com/) and fetched from the Meta Graph API.

## List Templates

```bash
GET /whatsapp/templates
```

Returns all message templates for the authenticated account.

### Authentication

Can be called with either:
- **Account API key** — templates for that account
- **Master API key** — requires passing the account context

<ParamField header="X-Api-Key" type="string" required>
  Account API key or Master API key
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum number of templates to return
</ParamField>

### Response

```json
{
  "data": [
    {
      "name": "hello_world",
      "status": "APPROVED",
      "category": "UTILITY",
      "language": "en_US",
      "components": [
        {
          "type": "BODY",
          "text": "Hello {{1}}! Welcome to our service."
        }
      ],
      "id": "123456789"
    }
  ]
}
```

### Template Statuses

| Status | Description |
|--------|-------------|
| `APPROVED` | Ready to use |
| `PENDING` | Under review by Meta |
| `REJECTED` | Not approved — check rejection reason in Meta Business Manager |

### Template Categories

| Category | Description |
|----------|-------------|
| `UTILITY` | Transaction updates, account notifications |
| `MARKETING` | Promotional messages, offers |
| `AUTHENTICATION` | OTP and verification codes |

## List Templates by Account (Admin)

```bash
GET /whatsapp/accounts/:id/templates
```

<ParamField header="X-Api-Key" type="string" required>
  Master API key
</ParamField>

<ParamField path="id" type="string" required>
  Account UUID
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum number of templates to return
</ParamField>

Returns templates for a specific account. Requires master API key.

### Response

Same format as the account-scoped endpoint above.
