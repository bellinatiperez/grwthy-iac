# Messages API

Send messages through the WhatsApp Cloud API. All messages are queued asynchronously for reliable delivery with automatic retries (3 attempts, exponential backoff).

## Authentication

Messages can be sent using either:
- **Account API key** ‚Äî the account is resolved from the key
- **Master API key** ‚Äî must include `accountId` in the request body

## Queue Response

All send endpoints return the same envelope:

```json
{
  "id": "message-log-uuid",
  "status": "queued",
  "accountId": "account-uuid",
  "to": "5511999999999",
  "type": "text"
}
```

The message is processed asynchronously. Track delivery status via [webhook events](/webhooks/events).

## Send Text

```bash
POST /whatsapp/messages/text
```

<ParamField body="to" type="string" required>
  Recipient phone number (with country code, no `+`)
</ParamField>

<ParamField body="body" type="string" required>
  Message text (max 4096 characters)
</ParamField>

<ParamField body="previewUrl" type="boolean">
  Enable URL link previews in the message
</ParamField>

<ParamField body="accountId" type="string">
  Account UUID (required when using master key)
</ParamField>

```bash
curl -X POST https://your-domain.com/whatsapp/messages/text \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: YOUR_ACCOUNT_API_KEY" \
  -d '{
    "to": "5511999999999",
    "body": "Hello from Grwthy!"
  }'
```

## Send Template

```bash
POST /whatsapp/messages/template
```

Templates must be pre-approved in your Meta Business account.

<ParamField body="to" type="string" required>
  Recipient phone number
</ParamField>

<ParamField body="templateName" type="string" required>
  Approved template name
</ParamField>

<ParamField body="languageCode" type="string" required>
  Template language code (e.g., `en_US`, `pt_BR`)
</ParamField>

<ParamField body="components" type="array">
  Template components with dynamic parameters
</ParamField>

<ParamField body="accountId" type="string">
  Account UUID (required when using master key)
</ParamField>

### Template Components

```json
{
  "to": "5511999999999",
  "templateName": "order_confirmation",
  "languageCode": "pt_BR",
  "components": [
    {
      "type": "body",
      "parameters": [
        { "type": "text", "text": "Jo√£o" },
        { "type": "text", "text": "#12345" }
      ]
    }
  ]
}
```

#### Component Fields

| Field | Type | Description |
|-------|------|-------------|
| `type` | `string` | `header`, `body`, or `button` |
| `sub_type` | `string` | Button sub-type (optional) |
| `index` | `number` | Button index (optional) |
| `parameters` | `array` | Parameter values |

#### Parameter Types

| Type | Description |
|------|-------------|
| `text` | Text substitution |
| `currency` | Currency value |
| `date_time` | Date/time value |
| `image` | Image media |
| `document` | Document media |
| `video` | Video media |

## Send Interactive Buttons

```bash
POST /whatsapp/messages/interactive/buttons
```

<ParamField body="to" type="string" required>
  Recipient phone number
</ParamField>

<ParamField body="body" type="string" required>
  Message body (max 1024 characters)
</ParamField>

<ParamField body="buttons" type="array" required>
  1 to 3 reply buttons
</ParamField>

<ParamField body="header" type="object">
  Optional header (`{ "type": "text", "text": "..." }`)
</ParamField>

<ParamField body="footer" type="string">
  Footer text (max 60 characters)
</ParamField>

### Button Object

| Field | Type | Description |
|-------|------|-------------|
| `type` | `string` | Must be `"reply"` |
| `reply.id` | `string` | Unique button identifier |
| `reply.title` | `string` | Button label (max 20 chars) |

```json
{
  "to": "5511999999999",
  "body": "How can we help?",
  "buttons": [
    { "type": "reply", "reply": { "id": "support", "title": "Support" } },
    { "type": "reply", "reply": { "id": "sales", "title": "Sales" } }
  ]
}
```

